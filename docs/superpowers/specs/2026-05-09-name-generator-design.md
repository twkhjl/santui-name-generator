# 假名產生器設計規格

## 目標

建立一個純前端網頁版假名產生器，能產生 126 個不重複的兩字繁體中文化名，並以固定 `14 x 9` 表格呈現，同時支援匯出為 `A4 直式 PDF`。

## 使用情境

- 使用者開啟頁面後，先由 `config.json` 載入預設說明文字與名字資料來源。
- 使用者可直接修改頁面上方說明文字。
- 使用者點擊「產生名字」後，系統產生並顯示 126 個不重複的兩字化名。
- 使用者點擊「匯出 PDF」後，系統將目前畫面中的說明文字與表格輸出成 PDF。

## 功能需求

### 1. 畫面結構

- 頁面上方有一個可編輯文字區塊，初始值來自 `config.json` 的 `defaultHeaderText`。
- 頁面中有一個「產生名字」按鈕。
- 頁面中有一個「匯出 PDF」按鈕。
- 頁面下方有固定 `14` 橫列、`9` 直行的表格，共 `126` 格。
- 每一格顯示一個兩字繁體中文化名。

### 2. 設定檔

設定檔檔名固定為 `config.json`，資料結構如下：

```json
{
  "defaultHeaderText": "這裡是上方說明文字",
  "fullNames": ["王明", "李安", "陳華"],
  "firstChars": ["王", "李", "陳", "林"],
  "secondChars": ["明", "安", "華", "婷"],
  "pdf": {
    "format": "a4",
    "orientation": "portrait",
    "marginMm": 10
  }
}
```

欄位定義：

- `defaultHeaderText`：頁面上方文字輸入區的預設內容。
- `fullNames`：完整兩字化名清單，系統優先使用。
- `firstChars`：產生化名第一字字池。
- `secondChars`：產生化名第二字字池。
- `pdf.marginMm`：PDF 頁面邊界，單位為毫米。

### 3. 名字產生規則

- 總共產生 `126` 個名字。
- 所有名字必須唯一，不可重複。
- 每個名字必須為兩個繁體中文字。
- 系統先使用 `fullNames`。
- `fullNames` 去重後若不足 `126`，再以 `firstChars x secondChars` 的所有可能組合補足。
- 補名時，先產生所有可用組合、排除已存在名字、隨機打亂，再取所需數量，避免固定順序導致結果偏斜。
- 若 `fullNames` 去重後，加上 `firstChars x secondChars` 的唯一組合總數仍小於 `126`，系統必須報錯，且不得產出不完整表格。

### 4. PDF 匯出規則

- 匯出格式固定為 `A4`、`直式`。
- PDF 內容需包含：
  - 當下畫面中可編輯文字區的內容
  - 當下表格中的 `126` 個名字
- 表格維持 `14 x 9` 版面。
- 每格文字置中顯示。
- PDF 需內嵌可正確顯示繁體中文的字型，例如 `Noto Sans TC` 或其他可合法使用字型。
- 預設匯出檔名為 `name-generator.pdf`。

## 非功能需求

### 1. 技術選型

採用純前端實作：

- `index.html`
- `style.css`
- `app.js`
- `config.json`

PDF 產出建議使用 `jsPDF`，並搭配內嵌中文字型。

### 2. 相容性

- 以現代桌面瀏覽器為主。
- 不要求舊版 IE 相容。

### 3. 維護性

- 設定資料與程式邏輯分離。
- 名字產生、設定驗證、PDF 匯出應拆成獨立函式，避免耦合。

## 錯誤處理

- `config.json` 載入失敗：顯示明確錯誤訊息，例如「設定檔讀取失敗」。
- `fullNames` 中非兩字項目：忽略該項，並顯示警告數量。
- `firstChars` 或 `secondChars` 為空，且 `fullNames` 不足 `126`：顯示錯誤，停止產生。
- 可用唯一名字總數小於 `126`：顯示「可用化名不足 126 個」。
- 尚未成功產生表格前，不可匯出 PDF；點擊匯出時需提示先產生名字。

## 資料流程

1. 頁面載入。
2. 讀取 `config.json`。
3. 驗證設定資料格式與可用名字總量。
4. 將 `defaultHeaderText` 填入可編輯文字區。
5. 使用者點擊「產生名字」。
6. 系統產出 126 個不重複名字並渲染到 `14 x 9` 表格。
7. 使用者點擊「匯出 PDF」。
8. 系統使用目前畫面內容輸出 `A4 直式 PDF`。

## 模組切分

### `index.html`

- 定義頁面骨架。
- 包含標題文字輸入區、操作按鈕、表格容器。

### `style.css`

- 控制頁面排版。
- 控制 `14 x 9` 表格樣式。
- 控制 A4 預覽區比例與列印視覺。

### `app.js`

- 載入並解析 `config.json`。
- 驗證設定。
- 產生不重複名字。
- 渲染表格。
- 匯出 PDF。

### `config.json`

- 提供預設文字。
- 提供完整名字清單。
- 提供第一字與第二字字池。
- 提供 PDF 頁面邊界等參數。

## 測試重點

- 成功載入合法 `config.json`。
- `fullNames` 足夠時，可直接產生 `126` 個不重複名字。
- `fullNames` 不足時，可正確以 `firstChars x secondChars` 補滿。
- 名字總數不足時，能正確阻止產生並提示錯誤。
- 表格固定為 `14 x 9`。
- 匯出 PDF 後，繁體中文可正確顯示，且版面維持 A4 直式。

## 範圍界線

本階段不包含：

- 後端服務
- 使用者上傳設定檔
- 名字規則語意分析
- 多語系介面
- 行動裝置專用優化
